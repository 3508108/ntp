# Правила для агентів
- Відповіді, документація, описи, підсумки та комунікація з користувачем мають бути лише українською.
- Уся розробка має виконуватись на Go або Node.js. Bash допускається тільки як допоміжний glue для systemd, деплою та коротких адміністративних команд.
- Для SSH і серверних доступів використовувати тільки RSA-ключі.
- Для серверних підключень завжди спочатку намагатись підключатись тільки як `root`. Якщо `root` недоступний, явно зафіксувати причину винятку перед використанням іншого користувача.
- Усі нові сервери, ключі, секрети, доступи, моніторинги й описи мають дублюватись у 1Password vault `auto`.
- 1Password має strict priority як джерело істини для доступів, секретів і описів інфраструктури.
- Неактуальні ключі потрібно переносити в архів у 1Password і видаляти з провайдерів/серверів після перевірки, що вони більше не використовуються.
